# E-commerce API Test Examples: Guest & User Checkout Flow

This file contains example API requests to test the complete e-commerce flow that supports both guest and authenticated users. The flow ensures orders are only created after successful payment and shipment.

## 🛒 New Flow Overview

1. **User selects products** - Browse and add items to cart
2. **Cart in session** - Cart stored in session for guests, database for users
3. **Login/Register (optional)** - If user chooses to login/register, migrate cart to database
4. **Create shipment** - Generate shipment through Bosta API
5. **Process payment** - Initiate payment through Paymob
6. **Create order** - Order is only created after BOTH shipment and payment are successful

## Prerequisites
- Run `php artisan db:seed` to populate the database with test data
- Replace `BASE_URL` with your API base URL (e.g., http://localhost:8000/api)
- No authentication required for cart operations (guest flow supported)
- Authentication optional for checkout flow

## Authentication

### Register a new user (optional)
```bash
curl -X POST BASE_URL/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "New User",
    "email": "newuser@example.com",
    "password": "password123",
    "password_confirmation": "password123"
  }'
```

### Login to get token
```bash
curl -X POST BASE_URL/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "test@example.com",
    "password": "password"
  }'
```

```json
{
```

```bash
      "street": "123 Main St",
```

## 🔧 Error Scenarios

### Cart Errors
- `422` - Product not available/insufficient stock
- `404` - Product not found

### Checkout Errors
- `422` - Cart empty, validation failed, shipment/payment failed
- `500` - Server error during checkout process

### Payment Errors
- `422` - Payment initiation failed
- `500` - Webhook processing error

## 💡 Key Features

✅ **Guest Checkout** - No authentication required for cart and checkout
✅ **Session Cart** - Cart stored in session for guests
✅ **Cart Migration** - Automatic migration to database on login
✅ **Shipment First** - Shipment created before payment
✅ **Payment Gateway** - Paymob integration with webhooks
✅ **Order Creation** - Only after successful payment AND shipment
✅ **Stock Management** - Automatic stock reduction/restoration
✅ **Error Handling** - Comprehensive error responses
✅ **Webhook Support** - Automated payment/shipment updates
